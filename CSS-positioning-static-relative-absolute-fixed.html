CSS positioning, static, relative, absolute, fixed

"Elements with a relative position value are like Clark Kent. They hide far greater powers than their static siblings." A List Apart

There is natural way CSS stacks information. These are types of boxes that stack in what's called normal flow. Giving elements a position delegates where it stacks. this can be done in 4 ways; static, relative, absolute, and fixed.

CSS positioning, static

Static positioning is described as the default setting. They stack in order of how they are written. This should be used for single columned layouts.

CSS positioning, relative

Relative positioning is the next step in positioning. You can add offset properties like top, left, right, and bottom.

CSS positioning, absolute

Absolute positioning provides more ultilty that relative positioning. With absolute positioning the box can be fluid. For example a box will stay flow in the browser when the viewport is resized.

CSS positioning, fixed

Fixed positioning is the final property used to adust CSS layout. The name says enough. The an element is fixed it stays in place, like when you scroll down a page.
